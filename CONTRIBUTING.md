# Contributing to CairnBooks

Thank you for your interest in contributing to CairnBooks! This document describes how to get involved and the guidelines we follow.

## Getting Started

1. **Fork** the repository and create your feature branch from `main`.
2. **Clone** your fork locally:
   ```bash
   git clone git@github.com:<your-username>/CairnBooks.git
   cd CairnBooks
   ```
3. **Install** any dependencies as described in the project README.

## How to Contribute

### Reporting Bugs

- Open a [GitHub Issue](../../issues) and use the **Bug report** template.
- Include steps to reproduce, expected behaviour, and actual behaviour.
- Attach screenshots or logs where relevant.

### Suggesting Features

- Open a [GitHub Issue](../../issues) and use the **Feature request** template.
- Describe the problem the feature solves and any alternative approaches you considered.

### Submitting Code

1. Make sure your changes are on a dedicated branch (`feature/my-feature` or `fix/my-fix`).
2. Keep commits focused and write clear commit messages.
3. Add or update tests for any new behaviour.
4. Run the existing test suite locally and ensure it passes before opening a pull request.
5. Open a **Pull Request** against `main` and fill out the PR description template.

## Code Style

- Follow the conventions already present in the codebase.
- Keep lines reasonably short and code readable.
- Comment non-obvious logic.
